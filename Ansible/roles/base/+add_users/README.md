# Role: add_users

Роль для добавления пользователей в систему с возможностью предоставления sudo привилегий.

## Описание

Эта роль позволяет:
- Создавать новых пользователей (только если они не существуют)
- Обновлять пароли существующих пользователей
- Автоматически создавать необходимые группы
- Назначать пользователей в группы
- Создавать домашние директории для пользователей
- Предоставлять sudo привилегии через группу sudo и файлы в `/etc/sudoers.d/`
- Безопасно управлять sudoers через `/etc/sudoers.d/`

## Переменные

### Обязательные переменные

- `users_to_add`: Список пользователей для добавления

### Опциональные переменные

- `debug_mode`: Включить/выключить отладочные сообщения (по умолчанию: `True`)

## Формат конфигурации пользователя

```yaml
users_to_add:
  - username: test_user_1
    password: "qwerty"
    groups: ["test_1", "test_2"]
    is_sudoers: True
    uid: null  # null для автоматического назначения
    gid: null  # null для автоматического назначения
    shell: /bin/bash
```

## Параметры пользователя

| Параметр | Описание | Обязательный | По умолчанию |
|----------|----------|--------------|--------------|
| `username` | Имя пользователя | Да | - |
| `password` | Пароль пользователя | Да | - |
| `groups` | Список групп | Нет | [] |
| `is_sudoers` | Предоставить sudo права | Нет | false |
| `uid` | UID пользователя | Нет | auto |
| `gid` | GID пользователя | Нет | auto |
| `shell` | Оболочка пользователя | Нет | /bin/bash |

## Пример использования

```yaml
- hosts: servers
  roles:
    - role: base/add_users
      vars:
        users_to_add:
          - username: admin
            password: "{{ vault_admin_password }}"
            groups: ["docker"]  # группа sudo добавляется автоматически при is_sudoers: true
            is_sudoers: true
            shell: /bin/bash
```

## Поведение роли

- **Создание пользователей**: Роль создает пользователей только если они не существуют в системе
- **Обновление паролей**: Пароли существующих пользователей обновляются при каждом запуске
- **Автоматическое создание групп**: Все группы, указанные в параметре `groups`, создаются автоматически
- **Sudo привилегии**: При `is_sudoers: true` пользователь автоматически добавляется в группу `sudo` и создается файл в `/etc/sudoers.d/`
- **Домашние директории**: Создаются автоматически для всех пользователей

## Безопасность

- Пароли хешируются с использованием SHA-512
- Sudo права предоставляются через `/etc/sudoers.d/` для лучшей безопасности
- Файлы sudoers имеют правильные права доступа (0440)
- Проверка корректности sudoers файлов через `visudo`
- Директория `/etc/sudoers.d/` создается с правами 0755 и принадлежностью root:root
