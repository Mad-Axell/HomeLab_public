# install_apts

Ansible роль для установки основных пакетов на системах Ubuntu/Debian с комплексной валидацией и возможностями отладки.

## Описание

Эта роль предоставляет надежное решение для установки основных пакетов на системах Debian/Ubuntu. Она включает валидацию параметров, комплексную обработку ошибок, подробный отладочный вывод и проверку успешной установки.

## Возможности

- ✅ **Валидация параметров**: Комплексная валидация всех параметров роли
- ✅ **Обработка ошибок**: Надежная обработка ошибок с механизмами повтора
- ✅ **Отладочный вывод**: Подробная отладочная информация при включении
- ✅ **Проверка системы**: Проверка системных предварительных условий и доступности пакетов
- ✅ **Проверка установки**: Проверка успешной установки пакетов
- ✅ **Гибкая конфигурация**: Поддержка основных и дополнительных пакетов
- ✅ **Управление кэшем**: Настраиваемое управление кэшем APT
- ✅ **Контроль обновлений**: Опциональные обновления системных пакетов

## Требования

- Ansible 2.9 или выше
- Системы на базе Debian/Ubuntu
- Менеджер пакетов APT
- Python 3 (для выполнения Ansible)

## Переменные роли

### Обязательные переменные

Нет - все переменные имеют разумные значения по умолчанию.

### Опциональные переменные

| Переменная | По умолчанию | Описание |
|------------|--------------|----------|
| `debug_mode` | `false` | Включить подробный отладочный вывод |
| `apt_cache_valid_time` | `86400` | Время валидности кэша APT в секундах (24 часа) |
| `apt_update_cache` | `true` | Обновлять ли кэш APT |
| `apt_upgrade_packages` | `true` | Обновлять ли системные пакеты |
| `apt_install_timeout` | `600` | Таймаут установки пакетов в секундах |
| `apt_install_recommends` | `true` | Устанавливать рекомендуемые пакеты |
| `apt_install_suggests` | `false` | Устанавливать предлагаемые пакеты |

### Списки пакетов

| Переменная | По умолчанию | Описание |
|------------|--------------|----------|
| `apt_packages` | См. defaults | Список основных пакетов для установки |
| `apt_optional_packages` | `[]` | Список дополнительных пакетов для установки |

## Основные пакеты по умолчанию

- `wget` - Загрузчик файлов из интернета
- `python3` - Интерпретатор Python 3
- `python3-pip` - Установщик пакетов Python
- `python3-requests` - HTTP библиотека для Python
- `build-essential` - Основные инструменты сборки
- `apt-transport-https` - HTTPS транспорт для APT
- `software-properties-common` - Управление свойствами ПО
- `ca-certificates` - Центры сертификации
- `openssh-client` - SSH клиент
- `iputils-ping` - Инструменты тестирования сети
- `dnsutils` - DNS утилиты

Нет.

## Пример плейбука

### Базовое использование

```yaml
---
- name: Установка основных пакетов
  hosts: all
  become: yes
  roles:
    - install_apts
```

### Расширенное использование с пользовательской конфигурацией

```yaml
---
- name: Установка пакетов с пользовательской конфигурацией
  hosts: all
  become: yes
  vars:
    debug_mode: true
    apt_update_cache: true
    apt_upgrade_packages: false
    apt_cache_valid_time: 3600
    apt_optional_packages:
      - vim
      - git
      - tree
      - jq
  roles:
    - install_apts
```

### Пропуск обновления пакетов

```yaml
---
- name: Установка пакетов без обновления системы
  hosts: all
  become: yes
  vars:
    apt_upgrade_packages: false
  roles:
    - install_apts
```

### Установка только определенных пакетов

```yaml
---
- name: Установка пользовательского списка пакетов
  hosts: all
  become: yes
  vars:
    apt_packages:
      - curl
      - wget
      - python3
      - git
    apt_optional_packages:
      - vim
      - htop
  roles:
    - install_apts
```

## Теги

Роль поддерживает следующие теги:

- `validation` - Запустить только валидацию параметров
- `always` - Всегда запускать (используется для валидации)

## Режим отладки

Включите режим отладки для просмотра подробной информации о процессе установки:

```yaml
vars:
  debug_mode: true
```

Отладочный вывод включает:
- Системную информацию
- Статус установки пакетов
- Результаты обновления кэша APT
- Результаты проверки установки
- Финальную сводку установки

## Обработка ошибок

Роль включает комплексную обработку ошибок:

- **Валидация параметров**: Проверяет все входные параметры перед выполнением
- **Системные предварительные условия**: Проверяет совместимость системы и доступность APT
- **Механизм повтора**: Повторяет установку пакетов до 3 раз с задержкой в 10 секунд
- **Проверка установки**: Проверяет, что все пакеты успешно установлены
- **Понятные сообщения об ошибках**: Предоставляет подробные сообщения об ошибках для устранения неполадок

## Валидация

Роль включает отдельный файл валидации (`tasks/validate.yml`), который:

- Проверяет типы и форматы параметров
- Проверяет форматы имен пакетов
- Убеждается, что булевы параметры правильно установлены
- Предоставляет подробную обратную связь по валидации

## Тестирование

Для тестирования роли:

```bash
# Тест с режимом отладки
ansible-playbook -i inventory playbook.yml -e "debug_mode=true"

# Тест только валидации
ansible-playbook -i inventory playbook.yml --tags validation

# Тест на определенных хостах
ansible-playbook -i inventory playbook.yml --limit ubuntu_servers
```

## Устранение неполадок

### Распространенные проблемы

1. **Не удается установить пакет**: Проверьте, правильно ли указано имя пакета и доступен ли он в репозиториях
2. **Проблемы с кэшем APT**: Попробуйте установить `apt_update_cache: true` и `apt_cache_valid_time: 0`
3. **Проблемы с правами доступа**: Убедитесь, что плейбук запускается с `become: yes`
4. **Проблемы с сетью**: Проверьте подключение к репозиториям пакетов

### Отладочная информация

Включите режим отладки для получения подробной информации:

```yaml
vars:
  debug_mode: true
```
