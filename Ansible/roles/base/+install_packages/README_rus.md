# install_packages

Корпоративная Ansible роль для установки основных пакетов на системах Ubuntu/Debian с комплексной валидацией, расширенными возможностями отладки и обработкой ошибок корпоративного уровня.

## Описание

Эта роль предоставляет надежное, корпоративное решение для установки основных пакетов на системах Debian/Ubuntu. Она включает комплексную валидацию параметров, расширенную обработку ошибок с механизмами повтора, подробный отладочный вывод, мониторинг системных ресурсов и тщательную проверку успешной установки. Роль разработана для производственных сред с расширенным логированием и возможностями мониторинга производительности.

## Ключевые возможности

- ✅ **Расширенная валидация параметров**: Комплексная валидация всех параметров роли с детальными сообщениями об ошибках
- ✅ **Корпоративная обработка ошибок**: Надежная обработка ошибок с механизмами повтора и комплексным анализом сбоев
- ✅ **Расширенный отладочный вывод**: Подробная отладочная информация включая системные ресурсы, доступность пакетов и метрики производительности
- ✅ **Проверка совместимости системы**: Тщательные проверки системных предварительных условий, совместимости ОС и доступности менеджера пакетов
- ✅ **Проверка установки**: Полная проверка успешной установки пакетов с пост-установочной валидацией
- ✅ **Гибкая конфигурация**: Поддержка основных и дополнительных пакетов с настраиваемыми политиками установки
- ✅ **Интеллектуальное управление кэшем**: Настраиваемое управление кэшем APT с периодами валидности и анализом репозиториев
- ✅ **Контроль обновлений**: Опциональные обновления системных пакетов с комплексным анализом воздействия
- ✅ **Мониторинг производительности**: Отслеживание времени выполнения и сбор метрик производительности
- ✅ **Мониторинг ресурсов**: Сбор информации о системных ресурсах для планирования мощностей
- ✅ **Анализ репозиториев**: Комплексный анализ доступности и конфигурации репозиториев

## Требования

- Ansible 2.9 или выше
- Системы на базе Debian/Ubuntu (Ubuntu 20.04+, Debian 11+)
- Менеджер пакетов APT
- Python 3 (для выполнения Ansible)
- Достаточное дисковое пространство для установки пакетов
- Сетевое подключение к репозиториям пакетов
- Соответствующие системные привилегии (доступ sudo/root)

## Переменные роли

### Обязательные переменные

Нет - все переменные имеют разумные значения по умолчанию.

### Опциональные переменные

| Переменная | По умолчанию | Описание |
|------------|--------------|----------|
| `debug_mode` | `false` | Включить комплексный отладочный вывод с системными ресурсами, метриками производительности и детальными логами операций |
| `apt_cache_valid_time` | `86400` | Время валидности кэша APT в секундах (24 часа) - контролирует когда требуется обновление кэша |
| `apt_update_cache` | `true` | Обновлять ли кэш APT перед операциями с пакетами |
| `apt_upgrade_packages` | `true` | Обновлять ли системные пакеты до последних версий |
| `apt_install_timeout` | `600` | Таймаут установки пакетов в секундах (диапазон 1-3600) |
| `apt_install_recommends` | `true` | Устанавливать рекомендуемые пакеты вместе с основными пакетами |
| `apt_install_suggests` | `false` | Устанавливать предлагаемые пакеты (не рекомендуется для продакшена) |

### Списки пакетов

| Переменная | По умолчанию | Описание |
|------------|--------------|----------|
| `apt_packages` | См. defaults | Список основных пакетов для установки |
| `apt_optional_packages` | `[]` | Список дополнительных пакетов для установки |

## Пример плейбука

### Базовое использование

```yaml
---
- name: Установка основных пакетов
  hosts: all
  become: yes
  roles:
    - install_packages
```

### Расширенное использование с пользовательской конфигурацией

```yaml
---
- name: Установка пакетов с пользовательской конфигурацией
  hosts: all
  become: yes
  vars:
    debug_mode: true
    apt_update_cache: true
    apt_upgrade_packages: false
    apt_cache_valid_time: 3600
    apt_optional_packages:
      - vim
      - git
      - tree
      - jq
  roles:
    - install_packages
```

### Пропуск обновления пакетов

```yaml
---
- name: Установка пакетов без обновления системы
  hosts: all
  become: yes
  vars:
    apt_upgrade_packages: false
  roles:
    - install_packages
```

### Установка только определенных пакетов

```yaml
---
- name: Установка пользовательского списка пакетов
  hosts: all
  become: yes
  vars:
    apt_packages:
      - curl
      - wget
      - python3
      - git
    apt_optional_packages:
      - vim
      - htop
  roles:
    - install_packages
```

## Корпоративная обработка ошибок

Роль включает комплексную, корпоративную обработку ошибок:

- **Расширенная валидация параметров**: Проверяет все входные параметры с детальной проверкой типов и диапазонов
- **Проверка системных предварительных условий**: Комплексные проверки совместимости системы, семейства ОС и доступности APT
- **Интеллектуальный механизм повтора**: Повторяет установку пакетов до 3 раз с задержкой в 10 секунд и экспоненциальным откатом
- **Проверка установки**: Полная проверка всех пакетов с пост-установочной валидацией
- **Детальные сообщения об ошибках**: Предоставляет комплексные сообщения об ошибках с руководством по устранению неполадок
- **Graceful обработка сбоев**: Использует блоки/спасение для структурированной обработки ошибок
- **Мониторинг ресурсов**: Мониторит системные ресурсы во время установки для предотвращения сбоев
- **Валидация репозиториев**: Проверяет доступность репозиториев перед попытками установки
- **Проверка доступности пакетов**: Валидирует доступность пакетов в репозиториях перед установкой

## Расширенная система валидации

Роль включает комплексную систему валидации (`tasks/validate.yml`), которая предоставляет:

### Валидация параметров
- **Валидация типов**: Проверяет типы параметров (строка, число, булево, список)
- **Валидация форматов**: Проверяет форматы и структуры параметров
- **Валидация диапазонов**: Проверяет числовые диапазоны (например, значения таймаута)
- **Валидация содержимого**: Проверяет имена пакетов и значения конфигурации

### Валидация пакетов
- **Формат имен пакетов**: Проверяет форматы имен пакетов и ограничения
- **Валидация списков пакетов**: Убеждается, что списки пакетов правильно отформатированы
- **Проверка доступности**: Проверяет доступность пакетов в репозиториях

### Валидация конфигурации
- **Валидация булевых параметров**: Убеждается, что булевы параметры правильно установлены
- **Валидация таймаута**: Проверяет значения таймаута в допустимых диапазонах
- **Конфигурация кэша**: Проверяет параметры, связанные с кэшем

### Комплексная обратная связь
- **Детальные сообщения об ошибках**: Предоставляет конкретные сообщения об ошибках для каждого сбоя валидации
- **Подтверждение успеха**: Подтверждает успешную валидацию всех параметров
- **Сводка валидации**: Отображает комплексные результаты валидации

## Устранение неполадок

### Распространенные проблемы

1. **Не удается установить пакет**: Проверьте, правильно ли указано имя пакета и доступен ли он в репозиториях
2. **Проблемы с кэшем APT**: Попробуйте установить `apt_update_cache: true` и `apt_cache_valid_time: 0`
3. **Проблемы с правами доступа**: Убедитесь, что плейбук запускается с `become: yes`
4. **Проблемы с сетью**: Проверьте подключение к репозиториям пакетов

Включите режим отладки для получения подробной информации:

```yaml
vars:
  debug_mode: true
```
