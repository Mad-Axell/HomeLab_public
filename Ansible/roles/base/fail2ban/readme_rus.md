# Роль Fail2Ban для Ansible

## Описание

Роль `base/fail2ban` предназначена для автоматической настройки и конфигурации системы предотвращения вторжений Fail2Ban в соответствии с лучшими практиками безопасности Ansible. Роль обеспечивает комплексную защиту SSH-сервера и других сервисов от брутфорс-атак и несанкционированных попыток доступа.

### Версия 2.0.0 - Улучшения

- ✅ **Комплексная валидация параметров** - все параметры проверяются перед применением
- ✅ **Расширенная отладочная информация** - подробные логи выполнения для диагностики
- ✅ **Улучшенные названия задач** - более описательные и информативные названия
- ✅ **Двуязычные комментарии** - все комментарии на русском и английском языках
- ✅ **Полная тегизация** - все задачи помечены тегами для выборочного выполнения
- ✅ **Обновленная документация** - подробные руководства и примеры использования

## Назначение

Fail2Ban — это фреймворк для предотвращения вторжений, который защищает серверы от брутфорс-атак. Данная роль предоставляет:

- **Защиту SSH**: Настраивает тюрьму для SSH с настраиваемыми параметрами
- **Интеграцию с UFW**: Автоматически блокирует IP-адреса через UFW firewall
- **Настраиваемое логирование**: Конфигурирует rsyslog для детального логирования SSH
- **Валидацию конфигурации**: Проверяет корректность всех параметров перед применением
- **Отладочную информацию**: Подробные логи выполнения для диагностики
- **Поддержку дополнительных сервисов**: Возможность настройки тюрем для nginx, apache и других сервисов

## Требования

### Системные требования
- **Ansible**: версия 2.9 или выше
- **ОС**: Debian-based Linux (Ubuntu, Debian)
- **Права доступа**: root или sudo привилегии
- **Firewall**: UFW (рекомендуется)

### Зависимости
- `fail2ban` - основная система защиты
- `rsyslog` - система логирования
- `iptables` - для работы с firewall

## Переменные роли

### Основная конфигурация

| Переменная | По умолчанию | Описание |
|------------|--------------|----------|
| `fail2ban_install` | `true` | Устанавливать ли Fail2Ban |
| `fail2ban_packages` | `['fail2ban', 'rsyslog', 'iptables']` | Необходимые пакеты |
| `debug_mode` | `true` | Включить отладочный вывод |

### Конфигурация SSH

| Переменная | По умолчанию | Описание |
|------------|--------------|----------|
| `ssh_port` | `22` | Номер порта SSH |
| `ssh_protocol` | `"tcp"` | Протокол SSH |

### Конфигурация тюрем

| Переменная | По умолчанию | Описание |
|------------|--------------|----------|
| `fail2ban_jails` | См. ниже | Основные тюрьмы Fail2Ban |
| `fail2ban_additional_jails` | См. ниже | Дополнительные тюрьмы |

#### Основные тюрьмы (fail2ban_jails)

```yaml
fail2ban_jails:
  ssh:
    enabled: true
    port: "{{ ssh_port }}"
    filter: sshd
    logpath: /var/log/sshd/sshd.log
    maxretry: 3
    findtime: 600
    bantime: 3600
    banaction: ufw
    ignoreip: "127.0.0.1/8 ::1"
  
  sshd:
    enabled: true
    port: "{{ ssh_port }}"
    filter: sshd
    logpath: /var/log/auth.log
    maxretry: 3
    findtime: 600
    bantime: 3600
    banaction: ufw
    ignoreip: "127.0.0.1/8 ::1"
```

#### Дополнительные тюрьмы (fail2ban_additional_jails)

```yaml
fail2ban_additional_jails:
  nginx-http-auth:
    enabled: false
    port: "http,https"
    filter: nginx-http-auth
    logpath: /var/log/nginx/error.log
    maxretry: 3
    findtime: 600
    bantime: 3600
    banaction: ufw
  
  apache-auth:
    enabled: false
    port: "http,https"
    filter: apache-auth
    logpath: /var/log/apache2/error.log
    maxretry: 3
    findtime: 600
    bantime: 3600
    banaction: ufw
```

### Глобальная конфигурация Fail2Ban

| Переменная | По умолчанию | Описание |
|------------|--------------|----------|
| `fail2ban_global_config` | См. ниже | Глобальные настройки Fail2Ban |

```yaml
fail2ban_global_config:
  loglevel: INFO
  logtarget: /var/log/fail2ban.log
  dbpurgeage: 86400
  socket: /var/run/fail2ban/fail2ban.sock
  pidfile: /var/run/fail2ban/fail2ban.pid
  dbfile: /var/lib/fail2ban/fail2ban.sqlite3
  banaction: ufw
  banaction_allports: ufw
```

| Переменная | По умолчанию | Описание |

ssh_logging:

## Примеры использования

### Базовое использование

```yaml
---
- hosts: servers
  become: true
  roles:
    - role: base/fail2ban
```

### Настройка с пользовательскими параметрами

```yaml
---
- hosts: servers
  become: true
  roles:
    - role: base/fail2ban
      vars:
        ssh_port: 2222
        debug_mode: true
        fail2ban_jails:
          ssh:
            enabled: true
            maxretry: 5
            findtime: 300
            bantime: 7200
            ignoreip: "127.0.0.1/8 ::1 192.168.1.0/24"
```

### Включение дополнительных тюрем

```yaml
---
- hosts: webservers
  become: true
  roles:
    - role: base/fail2ban
      vars:
        fail2ban_additional_jails:
          nginx-http-auth:
            enabled: true
            maxretry: 3
            findtime: 600
            bantime: 3600
          apache-auth:
            enabled: true
            maxretry: 3
            findtime: 600
            bantime: 3600
```

### Настройка с пользовательскими фильтрами

```yaml
---
- hosts: servers
  become: true
  roles:
    - role: base/fail2ban
      vars:
        fail2ban_custom_filters:
          custom-ssh:
            enabled: true
            port: "{{ ssh_port }}"
            filter: custom-ssh
            logpath: /var/log/custom-ssh.log
            maxretry: 3
            findtime: 600
            bantime: 3600
```

## Валидация конфигурации

Роль включает комплексную валидацию всех параметров:

1. **Обязательные переменные**: Проверка наличия всех необходимых переменных

## Управление службами

### Автоматический перезапуск

Роль автоматически перезапускает службы при изменении конфигурации:

- `fail2ban` - при изменении конфигурации тюрем или глобальных настроек
- `rsyslog` - при изменении конфигурации логирования
- `sshd` - при изменении настроек SSH

### Ручное управление

```bash
# Проверить статус Fail2Ban
sudo systemctl status fail2ban

# Перезапустить Fail2Ban
sudo systemctl restart fail2ban

# Проверить активные тюрьмы
sudo fail2ban-client status

# Проверить конкретную тюрьму
sudo fail2ban-client status sshd
```

## Мониторинг и логирование

### Логи Fail2Ban

```bash
# Основной лог Fail2Ban
sudo tail -f /var/log/fail2ban.log

# Лог SSH (если настроено отдельное логирование)
sudo tail -f /var/log/sshd/sshd.log

# Системный лог аутентификации
sudo tail -f /var/log/auth.log
```

### Мониторинг заблокированных IP

```bash
# Показать все заблокированные IP
sudo fail2ban-client status sshd

# Разблокировать конкретный IP
sudo fail2ban-client set sshd unbanip 192.168.1.100

# Разблокировать все IP
sudo fail2ban-client unban --all
```

## Безопасность

### Рекомендации по безопасности

1. **Настройка игнорируемых IP**: Добавьте доверенные сети в `fail2ban_ignoreip`
2. **Мониторинг логов**: Регулярно проверяйте логи на предмет подозрительной активности
3. **Обновления**: Регулярно обновляйте Fail2Ban и систему
4. **Резервные копии**: Используйте `backup_configuration: true`

### Настройка для высоконагруженных систем

- hosts: servers

## Устранение неполадок

### Частые проблемы

#### 1. Fail2Ban не запускается

```bash
# Проверить конфигурацию
sudo fail2ban-client -t

# Проверить логи
sudo journalctl -u fail2ban -f
```

#### 2. SSH тюрьма не работает

```bash
# Проверить путь к логам
sudo fail2ban-client status sshd

# Проверить права доступа к логам
sudo ls -la /var/log/auth.log
```

#### 3. UFW не блокирует IP

```bash
# Проверить статус UFW
sudo ufw status

# Проверить правила iptables
sudo iptables -L
```

### Диагностические команды

```bash
# Проверить конфигурацию Fail2Ban
sudo fail2ban-client -t

# Перезагрузить конфигурацию
sudo fail2ban-client reload

# Проверить активные фильтры
sudo fail2ban-client status

# Проверить конкретный фильтр
sudo fail2ban-client get sshd logpath
```
