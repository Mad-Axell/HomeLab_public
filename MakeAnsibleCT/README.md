# Автоматизация создания контейнера Proxmox LXC для Ansible и HomeLab

## Описание

`setup_container.sh` - это автоматизированный скрипт для создания и настройки Proxmox LXC контейнера, оптимизированного для работы с Ansible и HomeLab проектами. Скрипт автоматически выполняет все необходимые шаги от создания контейнера до настройки рабочего окружения.

## Возможности

### Автоматизация создания контейнера
- Автоматический поиск и загрузка последней версии OS Template
- Создание LXC контейнера с оптимальными параметрами
- Настройка сетевого интерфейса и IP адресации
- Автоматический запуск и инициализация

### Установка и настройка ПО
- Установка базовых системных пакетов
- Установка и настройка Ansible
- Настройка SSH доступа с пользовательскими учетными данными
- Создание структуры директорий Ansible

### Управление репозиториями
- Клонирование публичного репозитория HomeLab_public
- Клонирование приватного репозитория HomeLab_private с токеном GitHub
- Создание симлинков для ролей и конфигураций из обоих репозиториев

### Настройка Ansible
- Создание полной структуры директорий Ansible
- Настройка симлинков на роли, playbooks, переменные
- Автоматическая настройка прав доступа

## Требования

### Системные требования
- **ОС**: Proxmox VE (любая версия)
- **Права**: Root доступ
- **Архитектура**: AMD64
- **Свободное место**: Минимум 20GB для контейнера

### Программные требования
- Proxmox VE с поддержкой LXC
- Доступ к интернету для загрузки шаблонов и пакетов
- Git для клонирования репозиториев

## Установка и настройка

### 1. Клонирование репозитория
```bash
git clone https://github.com/Mad-Axell/HomeLab_private.git
cd HomeLab_private/MakeAnsibleCT
```

### 2. Настройка переменных
Отредактируйте файл `setup_container.sh` и настройте следующие параметры:

#### Основные параметры контейнера
```bash
CONTAINER_NAME="ansible-ct"        # Имя контейнера
CONTAINER_ID="200"                 # ID контейнера (должен быть уникальным)
CONTAINER_OS="debian"              # Предпочтительная ОС
CONTAINER_ARCH="amd64"             # Архитектура
```

#### Ресурсы контейнера
```bash
CONTAINER_CORES="2"                # Количество ядер CPU
CONTAINER_MEMORY="2048"            # RAM в MB
CONTAINER_SWAP="512"               # Swap в MB
CONTAINER_DISK_SIZE="20"           # Размер диска в GB
```

#### Сетевые настройки
```bash
CONTAINER_NETWORK="vmbr0"          # Сетевой мост
CONTAINER_IP="**.**.**.**/24"      # IP адрес контейнера
CONTAINER_GATEWAY="**.**.**.**"    # Шлюз по умолчанию
CONTAINER_DNS="8.8.8.8 8.8.4.4"    # DNS серверы
```

#### SSH настройки
```bash
SSH_USER="SSH_USER"                # Пользователь SSH
SSH_PASSWORD="SSH_PASSWORD"        # Пароль SSH
```

#### Git репозитории
```bash
GIT_REPO_USER="******"          # Пользователь GitHub
GIT_REPO_TOKEN="your_token_here"   # Personal Access Token для приватного репозитория
PUBLIC_GIT_DIR="HomeLab_public"    # Имя публичного репозитория
PRIVATE_GIT_DIR="HomeLab_private"  # Имя приватного репозитория
```

### 3. Создание GitHub токена
Для доступа к приватному репозиторию создайте Personal Access Token:

1. Перейдите в GitHub → Settings → Developer settings → Personal access tokens
2. Создайте новый токен с правами `repo`
3. Скопируйте токен в переменную `GIT_REPO_TOKEN`

### 4. Запуск скрипта
```bash
sudo ./setup_container.sh
```

## Структура создаваемого окружения

После выполнения скрипта в контейнере будет создана следующая структура:

```
/etc/
├── ansible/
│   ├── roles/          # Симлинки на роли из репозиториев
│   ├── group_vars/     # Симлинки на group_vars из приватного репозитория
│   ├── host_vars/      # Симлинки на host_vars из приватного репозитория
│   ├── Keys/           # Симлинки на ключи из приватного репозитория
│   ├── playbooks/      # Симлинки на playbooks из приватного репозитория
│   └── vars/           # Симлинки на переменные из приватного репозитория
└── home_automate/
    ├── HomeLab_public/ # Клонированный публичный репозиторий
    └── HomeLab_private/ # Клонированный приватный репозиторий
```

## Функции скрипта

### Основные функции
- `check_variables()` - Проверка корректности переменных
- `check_proxmox()` - Проверка окружения Proxmox
- `check_lxc()` - Проверка LXC окружения
- `find_latest_os_template()` - Поиск последнего OS Template
- `create_container()` - Создание LXC контейнера
- `install_base_packages()` - Установка базовых пакетов
- `install_ansible()` - Установка Ansible
- `setup_user_access()` - Настройка SSH доступа

### Специализированные функции
- `setup_home_automation()` - Клонирование репозиториев
- `setup_ansible_config()` - Настройка Ansible и создание симлинков
- `show_access_info()` - Вывод информации о доступе

## Устранение неполадок

### Частые проблемы

#### 1. Ошибка "pveversion не найден"
**Решение**: Убедитесь, что скрипт запущен на Proxmox сервере

#### 2. Ошибка клонирования приватного репозитория
**Решение**: 
- Проверьте правильность GitHub токена
- Убедитесь, что токен имеет права `repo`
- Проверьте доступность репозитория

#### 3. Ошибка создания контейнера
**Решение**:
- Проверьте уникальность CONTAINER_ID
- Убедитесь в наличии свободного места
- Проверьте сетевые настройки

#### 4. Ошибка установки пакетов
**Решение**:
- Проверьте подключение к интернету
- Убедитесь в доступности репозиториев пакетов

### Диагностика
```bash
# Проверка статуса контейнера
pct status 200

# Подключение к контейнеру
pct enter 200

# Просмотр логов
pct exec 200 -- journalctl -f

# Проверка сетевого подключения
pct exec 200 -- ping -c 3 8.8.8.8
```

## Производительность

### Оптимизация ресурсов
- **CPU**: 2 ядра достаточно для большинства задач Ansible
- **RAM**: 2GB обеспечивает комфортную работу
- **Диск**: 20GB для базовой установки, увеличьте при необходимости
- **Swap**: 512MB для стабильности

### Масштабирование
Для увеличения производительности отредактируйте:
```bash
CONTAINER_CORES="4"      # Больше ядер
CONTAINER_MEMORY="4096"  # Больше RAM
CONTAINER_DISK_SIZE="50" # Больше места на диске
```
